Sobre o Projeto

lemmAIngs e uma aplicacao de spec-driven development onde especificacoes se tornam artefatos executaveis que geram implementacoes funcionais. Em vez de "vibe coding", voce descreve o que quer construir e agentes de IA geram a especificacao, o design pattern, as tarefas e o codigo.

Inspirado em: GitHub spec-kit + LangChain deepagents

Instalacao

Passo 1 — Clonar ou copiar o projeto

bash # clonar o repositorio
git clone <url-do-repositorio> lemmAIngs
cd lemmAIngs

Passo 2 — Criar ambiente virtual (recomendado)

bash # Windows
python -m venv .venv
.venv\Scripts\activate

# Linux / macOS
python3 -m venv .venv
source .venv/bin/activate

Passo 3 — Instalar dependencias

bash pip install -r requirements.txt
Nota: O requirements.txt inclui o pacote deepagents que sera instalado diretamente do repositorio GitHub. Certifique-se de ter conexao com a internet.

Configuracao (.env)

Edite o arquivo .env na raiz do projeto:

.env OPENAI_API_KEY=sk-sua-chave-aqui
MODEL_NAME=gpt-4.1
APP_NAME=lemmAIngs
APP_HOST=0.0.0.0
APP_PORT=8000
Variavel Obrigatorio Descricao
OPENAI_API_KEYSimSua chave da API OpenAI
MODEL_NAMENaoModelo a usar (default: gpt-4o)
APP_NAMENaoNome exibido na app
APP_HOSTNaoHost do servidor (default: 0.0.0.0)
APP_PORTNaoPorta do servidor (default: 8000)
Seguranca: Nunca commite o arquivo .env no repositorio. Ele ja esta incluido no .gitignore.

bash python run.py

Guia de Uso — 5 Fases

1

Especificar (ESPEC.md)

Descreva O QUE voce quer construir e POR QUE.

Como usar:

  1. Clique em Novo Projeto ou selecione um existente
  2. Preencha o nome e a descricao do projeto
  3. Clique em Criar e Especificar
  4. O agente gerara o ESPEC.md completo com: visao geral, personas, jornadas, requisitos funcionais e nao-funcionais, criterios de aceite e escopo
Dica: Quanto mais detalhada sua descricao, melhor sera a especificacao gerada. Inclua o publico-alvo, problema a resolver e funcionalidades principais.
2

Design Pattern (DP.md)

Defina COMO sera construido e COM QUE tecnologias.

Como usar:

  1. (Opcional) Selecione um Design Pattern de referencia no dropdown — estes sao DPs cadastrados previamente via upload de PDF/DOCX
  2. Preencha a stack desejada (ex: Python, FastAPI, React, PostgreSQL)
  3. Adicione restricoes tecnicas se houver (ex: "deve rodar em Docker", "sem banco relacional")
  4. Clique em Gerar DP.md
  5. O agente gerara o plano tecnico com: arquitetura, stack, estrutura de diretorios, design patterns, modelagem de dados, APIs e estrategia de testes
Design Pattern de referencia: Se voce selecionou um DP previamente cadastrado, o agente usara aquele documento como base e adaptara ao seu projeto. Veja Catalogo de DPs para saber como cadastrar.
3

Tarefas (TASKS.md)

Quebre a especificacao em tarefas independentes e testaveis.

Como usar:

  1. Com ESPEC.md e DP.md gerados, clique na fase 3. Tarefas
  2. Clique em Gerar TASKS.md
  3. O agente analisa ambos artefatos e gera uma lista ordenada de tarefas, cada uma com: ID, titulo, descricao, dependencias, criterio de aceite e estimativa (P/M/G)
4

Implementar

O agente gera o codigo seguindo todos os artefatos.

Como usar:

  1. Clique na fase 4. Implementar
  2. (Opcional) Informe o numero da tarefa para implementar apenas uma tarefa especifica. Deixe vazio para implementar todas.
  3. Clique em Implementar
  4. O agente gerara IMPLEMENTATION.md com blocos de codigo organizados por arquivo
Dica: Para projetos grandes, implemente uma tarefa por vez para manter a qualidade e poder revisar cada entrega.
5

Download (.zip)

Baixe o projeto completo com todos os artefatos e codigo.

Como usar:

  1. Apos a fase 4, clique na fase 5. Download
  2. Verifique o checklist de artefatos disponiveis
  3. Clique em Baixar {nome-do-projeto}.zip

Catalogo de Design Patterns (Upload PDF/DOCX)

O catalogo permite cadastrar design patterns que voce ja possui como documentos PDF ou DOCX. Eles ficam disponiveis para selecao na fase 2 de qualquer projeto.

Como cadastrar um Design Pattern:

  1. Clique no botao DP Catalog no canto superior direito do header
  2. No modal, arraste um arquivo PDF ou DOCX para a area de upload, ou clique para selecionar
  3. De um nome para o design pattern (ex: "Clean Architecture", "Hexagonal Pattern")
  4. Clique em Enviar
  5. O sistema extrai o texto do documento e o armazena no catalogo

Como usar um DP cadastrado:

  1. Na fase 2 (Design Pattern) de qualquer projeto, veja o dropdown "Design Pattern de referencia"
  2. Selecione o DP desejado — um preview do conteudo aparecera abaixo
  3. Ao clicar em "Gerar DP.md", o agente usara aquele documento como base
Formatos suportados: PDF (.pdf) e Word (.docx, .doc). Tamanho maximo: 20MB.
Gerenciamento: Voce pode remover DPs cadastrados clicando no icone de lixeira dentro do catalogo.

Editar Artefatos

Todos os artefatos gerados (ESPEC.md, DP.md, TASKS.md, IMPLEMENTATION.md) podem ser editados manualmente:

  1. Navegue ate a fase do artefato que deseja editar
  2. Clique no icone de lapis (editar) no canto superior direito do painel de saida
  3. O painel muda para modo de edicao em texto puro (Markdown)
  4. Faca suas alteracoes e clique em Salvar

Referencia da API

Todos os endpoints estao sob /api. A documentacao interativa esta em /docs (Swagger UI).

Metodo Rota Descricao
GET/api/projectsLista todos os projetos
GET/api/projects/{name}Status e artefatos de um projeto
POST/api/specifyFase 1 — Gera ESPEC.md
POST/api/planFase 2 — Gera DP.md (aceita dp_template_id)
POST/api/tasksFase 3 — Gera TASKS.md
POST/api/implementFase 4 — Gera IMPLEMENTATION.md
PUT/api/projects/{name}/{artifact}Edita artefato manualmente
GET/api/projects/{name}/downloadFase 5 — Download .zip
GET/api/dp-catalogLista DPs cadastrados
POST/api/dp-catalog/uploadUpload PDF/DOCX como DP
GET/api/dp-catalog/{id}Conteudo extraido de um DP
DEL/api/dp-catalog/{id}Remove um DP do catalogo

Troubleshooting

Erro: "OPENAI_API_KEY not set" ou respostas vazias

A chave da API nao esta configurada ou e invalida.

fix # Verifique se o .env existe e contem a chave
cat .env

# A chave deve comecar com "sk-"
OPENAI_API_KEY=sk-...

Erro: "ModuleNotFoundError: No module named 'xxx'"

Dependencias nao instaladas ou ambiente virtual nao ativado.

fix # Ative o ambiente virtual
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # Linux/Mac

# Reinstale as dependencias
pip install -r requirements.txt

Erro: "Execute /specify antes de /plan"

As fases sao sequenciais. Voce precisa completar a fase anterior antes de avancar.

Ordem obrigatoria: Especificar → Design Pattern → Tarefas → Implementar → Download

Erro: Upload de PDF/DOCX falha

Verifique o formato e tamanho do arquivo.

Formatos aceitos: .pdf, .docx, .doc
Tamanho maximo: 20MB
Nota: PDFs escaneados (imagens) podem nao ter texto extraivel. Use PDFs com texto selecionavel.

Porta 8000 ja em uso

Outra aplicacao esta usando a porta.

fix # Altere a porta no .env
APP_PORT=8080

# Ou encerre o processo na porta 8000
# Windows: netstat -ano | findstr :8000
# Linux:  lsof -i :8000
